Integration Guide
Connect Your Trained Model
Step-by-step guide to connect your .pct model file (1.2 GB) to this website via a FastAPI backend.
1

Install Required Python Packages

Open a terminal and install the necessary libraries. Make sure Python 3.9+ is installed.

pip install fastapi uvicorn torch torchaudio
pip install transformers soundfile librosa python-multipart
2

Place Your Model File

Create a folder called backend/ and put your .pct file inside it:

your-project/
├── backend/
│   ├── model.pct          ← your trained model file
│   └── app.py             ← we create this next
└── speechsense.html       ← this website file
Note: Your .pct file is a PyTorch checkpoint saved with torch.save(). It will be loaded using torch.load().
3

Create the FastAPI Backend — app.py

Create a file called app.py inside your backend/ folder with this code:

# backend/app.py
import torch, torchaudio, io
from fastapi import FastAPI, UploadFile, File
from fastapi.middleware.cors import CORSMiddleware
from fastapi.responses import JSONResponse

app = FastAPI()

# Allow the website to call this backend
app.add_middleware(CORSMiddleware,
    allow_origins=["*"], allow_methods=["*"], allow_headers=["*"])

# ── Load your .pct model once at startup ──
MODEL_PATH = "model.pct"
checkpoint = torch.load(MODEL_PATH, map_location="cpu")

# Adapt these lines to match how YOUR model was saved
model = checkpoint.get("model") or checkpoint
model.eval()

# ── Analyse endpoint ──
@app.post("/api/analyze")
async def analyze(audio: UploadFile = File(...)):
    data = await audio.read()
    waveform, sr = torchaudio.load(io.BytesIO(data))

    # Resample to 16 kHz if needed
    if sr != 16000:
        waveform = torchaudio.functional.resample(waveform, sr, 16000)

    # Run your model — adjust to match your model's API
    with torch.no_grad():
        output = model(waveform)

    # Return results — adapt field names to your model output
    return JSONResponse({
        "transcript": output.get("transcript", ""),
        "phoneme_errors": output.get("errors", []),
        "accuracy": output.get("accuracy", 0),
        "per": output.get("per", 0)
    })
4

Run the Backend Server

Navigate into the backend/ folder and start the server:

cd backend
uvicorn app:app --reload --host 0.0.0.0 --port 8000

You should see: Uvicorn running on http://0.0.0.0:8000

Large model tip: Loading 1.2 GB may take 10–20 seconds on first startup. This is normal — it only loads once.
5

Connect the Website to Your Backend

Open speechsense.html in a text editor and find the runAnalysis() function. Replace the setTimeout demo block with a real API call:

// Find this in the <script> section and replace:
async function runAnalysis() {
  const file = document.getElementById('fileIn').files[0];
  if (!file) { alert('Please upload or record audio first.'); return; }

  const form = new FormData();
  form.append('audio', file);

  setLoading(true);
  const res = await fetch('http://localhost:8000/api/analyze', {
    method: 'POST', body: form
  });
  const data = await res.json();
  setLoading(false);
  renderResults(data);   // already wired up in the code
}
Expected JSON response format from your backend:
{ "transcript": "The rabbit ran...", "phoneme_errors": [{"word":"rabbit","phoneme":"/r/","type":"substitution","produced":"/w/","accuracy":52}], "accuracy": 87, "per": 7.69 }
6

Open the Website in Your Browser

Simply open speechsense.html directly in Chrome or Firefox. Make sure your backend is running at localhost:8000.

# Option A — open the file directly:
open speechsense.html        # macOS
start speechsense.html       # Windows

# Option B — serve via Python (avoids CORS issues):
python -m http.server 3000
# Then open http://localhost:3000/speechsense.html
Deployment: To make it accessible over a network (clinic or home), deploy the FastAPI backend on a server (e.g. EC2, Render, Railway) and update the URL in runAnalysis() from localhost:8000 to your server address.
✓

You're Done!

Go to the Analyse page, upload or record speech, and click Run Phoneme Analysis. Results, feedback, and the report will populate automatically from your model's output.
